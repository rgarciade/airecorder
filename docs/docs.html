Guía de IA Local

Aprende cómo configurar tu propio motor de inteligencia artificial privado para usarlo con AIRecorder sin compartir tus datos con nadie.

¿Qué son Ollama y LM Studio?

Para que AIRecorder pueda analizar tus transcripciones, resumir reuniones y responder preguntas sin depender de la nube, necesitamos un "motor" que ejecute los modelos de IA en tu propio ordenador. Existen dos herramientas líderes para esto:

Ollama

Es la opción más sencilla y la que recomendamos por defecto. Se ejecuta en segundo plano como un servicio y te permite descargar y correr modelos usando comandos de terminal muy simples.

  • ✅ Instalación en 1 clic
  • ✅ Integración perfecta con AIRecorder
  • ✅ Soporta modelos de chat y de embedding

LM Studio

Es una aplicación con interfaz gráfica (GUI) muy completa. Es ideal si te gusta explorar miles de modelos en HuggingFace, ajustar parámetros técnicos y ver gráficos de consumo de CPU/RAM.

  • ✅ Interfaz visual atractiva
  • ✅ Servidor compatible con la API de OpenAI
  • ✅ Control total sobre la memoria RAM (GGUF)

¿Cómo instalarlos?

Instalación de Ollama (Recomendado)

1. Visita ollama.com/download y descarga el instalador para tu sistema (Windows, Mac o Linux).

2. Ejecuta el archivo descargado e instala la aplicación.

3. Abre tu terminal (Símbolo del sistema en Windows o Terminal en Mac) y ejecuta este comando para descargar el modelo de embedding necesario para AIRecorder:

ollama pull nomic-embed-text

4. Luego, descarga un modelo de chat (por ejemplo, Llama 3):

ollama pull llama3.2
¡Listo! Abre AIRecorder, selecciona Ollama en la configuración y la app lo detectará automáticamente.

Instalación de LM Studio

1. Visita lmstudio.ai y descarga la aplicación.

2. Abre LM Studio. En la barra de búsqueda superior, busca y descarga un modelo de lenguaje (ej. Llama-3.2-3B-Instruct-GGUF).

3. Busca y descarga también un modelo de text-embedding (ej. nomic-embed-text-v1.5.Q8_0.gguf).

4. Ve a la pestaña de Local Server (icono de red en la barra izquierda).

5. Asegúrate de habilitar el modelo de Embedding y cargar tu modelo de lenguaje. Luego pulsa Start Server.

¡Listo! En AIRecorder, selecciona LM Studio y asegúrate de que la URL coincide (suele ser http://localhost:1234/v1).

Modelos Generativos vs. Embedding

AIRecorder utiliza dos tipos de modelos de IA que trabajan en equipo para ofrecerte una experiencia completa. A esta técnica se la conoce como sistema RAG (Retrieval-Augmented Generation).

1. Modelos Generativos (Chat)

Son el "cerebro" que habla contigo. Entienden instrucciones, redactan resúmenes, corrigen ortografía y generan respuestas legibles.

Se usan para:
  • • Crear el resumen automático de una reunión.
  • • Generar las tareas pendientes (Action Items).
  • • Responder a tus preguntas en el chat.

Recomendados: Llama 3 (8B), Qwen 2.5, DeepSeek R1, Gemma 2.

2. Modelos de Embedding

Son modelos "traductores". Convierten párrafos de texto en largas listas de números (vectores). No saben hablar, solo categorizar conceptos.

Se usan para:
  • • Leer tus transcripciones y guardarlas en la Base de Datos.
  • • Encontrar rápidamente de qué se habló en qué segundo.

Recomendados: nomic-embed-text, mxbai-embed-large.

La Ventana de Contexto

La ventana de contexto es la "memoria a corto plazo" de la IA. Es la cantidad máxima de palabras (o "tokens") que el modelo puede retener al mismo tiempo mientras te responde o resume un texto.

¿Cómo afecta al rendimiento?

Un contexto más grande permite a la IA analizar reuniones largas de una sola vez. Sin embargo, aumentar el contexto exige exponencialmente más memoria RAM. Si el contexto supera tu memoria disponible, tu ordenador se ralentizará enormemente.

Configuración en AIRecorder

En Ajustes > Agentes IA puedes definir este límite:

  • 4096 - Ordenadores básicos (8GB RAM)
  • 8192 - Recomendado (16GB RAM)
  • 32768+ - Equipos potentes (Macs M-series o GPUs)

💡 El sistema RAG al rescate

No te preocupes si no puedes poner un contexto muy alto. Cuando haces una pregunta en el chat, AIRecorder no le envía toda la reunión a la IA (lo que saturaría el contexto). En su lugar, usa el modelo de embedding para buscar los 5 párrafos más relevantes y solo le envía esos trozos. ¡Así puedes consultar reuniones de horas usando muy poca RAM!
