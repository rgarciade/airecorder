Guia de Usuario

Como grabar una sesion

AIRecorder te permite grabar el audio de tu microfono o de un dispositivo de audio virtual (como BlackHole en Mac) para capturar reuniones, clases o cualquier conversacion. Para empezar:

1. Selecciona la fuente

En la pantalla principal, elige entre microfono (tu voz) o audio del sistema (lo que escuchan tus altavoces). Para reuniones online, recomendamos audio del sistema.

2. Pulsa Grabar

Haz clic en el boton Grabar. Veras un indicador visual de que la grabacion esta activa. Puedes pausar y reanudar en cualquier momento.

Como ver transcripciones

Cuando detienes una grabacion, AIRecorder procesa el audio automaticamente usando Whisper (local o en la nube, segun tu configuracion). La transcripcion aparece en la vista principal con marcas de tiempo, texto y, si esta activada, identificacion de hablantes.

Como usar el chat con IA

Cada transcripcion tiene un panel de chat donde puedes hacer preguntas sobre el contenido de la reunion. La IA usa el sistema RAG para buscar los fragmentos mas relevantes y responderte con contexto preciso. Puedes preguntar cosas como "Que tareas se asignaron?" o "Quien hablo sobre el presupuesto?".

Como exportar

Desde la vista de transcripcion, puedes exportar el contenido en varios formatos: texto plano, Markdown, o PDF. Los analisis de IA (resumen, puntos clave, tareas) tambien se incluyen en la exportacion.

Como organizar en proyectos

AIRecorder te permite agrupar transcripciones en proyectos. Crea un proyecto desde la seccion de proyectos en la barra lateral, y luego asigna cada grabacion al proyecto correspondiente. Esto facilita buscar y filtrar contenido por contexto (trabajo, estudio, personal, etc.).

Gestionar hablantes

Con la diarizacion activa, AIRecorder identifica automaticamente los diferentes hablantes. Desde la vista de transcripcion puedes asignar nombres reales a cada perfil detectado: el sistema guarda el embedding de voz asociado y reconoce a esa persona automaticamente en grabaciones futuras.

Asignar nombre

Haz clic en el identificador del hablante en la transcripcion y escribe su nombre. AIRecorder vincula el embedding de voz al perfil y lo aplica automaticamente en futuras grabaciones.

Fusionar hablantes

Si la diarizacion ha dividido a la misma persona en varios perfiles, ve a la seccion Hablantes en la barra lateral y usa la opcion de fusion para combinarlos en un unico perfil sin perder el historial.

Arquitectura Tecnica

AIRecorder esta construida sobre cuatro pilares: Electron proporciona la ventana de escritorio, React renderiza la interfaz de usuario, Python procesa el audio y genera transcripciones, y SQLite almacena todos los datos localmente.

Flujo de comunicacion (Main → IPC → Renderer)

La aplicacion sigue el modelo de procesos de Electron: el main process (electron/main.js) gestiona las ventanas, el sistema de archivos y los procesos de Python. El renderer process (la app React) se comunica con el main a traves de IPC (Inter-Process Communication). El archivo electron/preload.js expone de forma segura los canales IPC al renderer mediante contextBridge.

Canales IPC principales:
  • recording:start — Inicia la captura de audio
  • recording:stop — Detiene y guarda el archivo WAV
  • transcription:process — Lanza el proceso Python de Whisper
  • ai:analyze — Envia texto al proveedor de IA para analisis
  • db:* — Operaciones de lectura/escritura en SQLite

Base de datos (better-sqlite3)

Los metadatos de transcripciones, proyectos, hablantes y configuraciones se almacenan en una base de datos SQLite local gestionada por better-sqlite3. Es una base de datos embebida que no requiere servidor: todo vive en un unico archivo .db en el directorio de datos de la aplicacion.

Transcription Manager

El modulo electron/transcriptionManager.js orquesta el pipeline de transcripcion: recibe el archivo WAV, invoca el script Python (python/audio_sync_analyzer.py), captura la salida JSON con los segmentos transcritos, y los almacena en la base de datos. Si la diarizacion esta activada, el mismo proceso Python ejecuta pyannote para identificar hablantes.

Proveedores de IA

AIRecorder soporta multiples proveedores de IA para que elijas el que mejor se adapte a tus necesidades: privacidad total con modelos locales, o potencia ilimitada con servicios en la nube.

Ollama (Local)

La opcion recomendada para uso local. Se ejecuta como servicio en segundo plano. Soporta modelos de chat y embedding. Ideal para: privacidad maxima, sin coste, sin internet.

LM Studio (Local)

Aplicacion con interfaz grafica. Compatible con la API de OpenAI. Ideal para: explorar muchos modelos, control fino de parametros, formato GGUF.

Gemini / DeepSeek / Kimi (Nube)

Servicios en la nube con modelos de alta capacidad. Ideal para: reuniones muy largas, analisis complejos, cuando no tienes hardware potente.

Como configurar

Ve a Ajustes > Agentes IA en la barra lateral. Selecciona tu proveedor preferido, introduce la URL del endpoint (para locales) o la API key (para nube), y elige los modelos de chat y embedding. Pulsa Verificar modelo para confirmar que la conexion funciona.

Guia detallada de configuracion

Para instrucciones paso a paso con capturas de pantalla, consulta la Guia de IA Local.

AIRecorder detecta automaticamente Ollama y LM Studio si estan corriendo en localhost.

Sistema RAG

RAG (Retrieval-Augmented Generation) es la tecnica que permite a la IA encontrar informacion relevante en tus transcripciones sin tener que leer todo el texto cada vez. En lugar de buscar palabras exactas, busca conceptos similares. Si preguntas sobre "presupuesto", la IA encontrara fragmentos que hablen de "costes", "gastos" o "financiacion" aunque no usen esa palabra exacta.

Como funciona tecnicamente

1. Embeddings (vectores numericos)

Cada fragmento de transcripcion se convierte en un vector de numeros usando un modelo de embedding. Estos vectores representan el significado semantico del texto. Dos textos con significado similar tendran vectores cercanos en el espacio matematico.

2. LanceDB (base de datos vectorial)

Los vectores se almacenan en LanceDB, una base de datos vectorial embebida que permite busquedas rapidas por similitud. No requiere servidor externo: funciona localmente como un archivo mas.

3. Chunking (segmentacion)

Las transcripciones se dividen en fragmentos (chunks) de tamano manejable antes de generar los embeddings. Esto asegura que cada busqueda devuelva fragmentos precisos y contextualizados, no parrafos gigantes.

4. Como afecta al chat

Cuando haces una pregunta en el chat, AIRecorder: (a) convierte tu pregunta en un vector, (b) busca los 5 fragmentos mas similares en LanceDB, (c) envia esos fragmentos como contexto a la IA junto con tu pregunta, y (d) devuelve la respuesta generada. Todo esto ocurre en menos de un segundo.

Importante sobre el modelo de embedding

El modelo de embedding debe coincidir entre la indexacion y la busqueda. Si cambias de modelo, necesitas re-indexar tus transcripciones. AIRecorder usa nomic-embed-text como modelo por defecto.

Prompts de IA

Los prompts son las instrucciones que le damos a la IA para que sepa que hacer con el texto de tus transcripciones. La calidad del prompt determina directamente la calidad de la respuesta. AIRecorder usa prompts optimizados para cada tipo de analisis.

Resumen

Genera un resumen conciso de la reunion. La IA identifica los temas principales discutidos, las decisiones tomadas y el contexto general. Util para tener una vision rapida sin leer toda la transcripcion.

Puntos clave

Extrae los puntos mas importantes de la conversacion en formato de lista. Ideal para revisar rapidamente los aspectos mas relevantes de una reunion larga.

Tareas pendientes

Identifica las acciones comprometidas, quien las asumio y cualquier plazo mencionado. Genera una lista de action items estructurada que puedes usar para dar seguimiento.

Chat libre

Responde preguntas especificas sobre el contenido de la transcripcion usando el contexto RAG. Puedes preguntar sobre cualquier aspecto de la reunion y la IA buscara la informacion relevante.

Prompts personalizados

Puedes personalizar los prompts desde Ajustes > Agentes IA > Prompts personalizados. Ahí encontrarás campos para modificar las instrucciones que recibe la IA según el tipo de analisis (resumen, puntos clave, tareas, chat). Los cambios se aplican inmediatamente sin necesidad de reiniciar la app.

Diarizacion

La diarizacion es el proceso de identificar quien hablo en cada momento de una grabacion. AIRecorder intenta distinguir entre diferentes hablantes y asignarles un nombre o identificador, para que puedas saber facilmente quien dijo que en la transcripcion.

Como funciona tecnicamente

1. pyannote.audio

AIRecorder utiliza pyannote.audio, una libreria de Python basada en redes neuronales, para detectar los segmentos de habla y agruparlos por hablante. El modelo pre-entrenado analiza las caracteristicas acusticas de la voz.

2. Embeddings de hablante

Cada segmento de voz se convierte en un embedding de hablante (un vector que representa las caracteristicas unicas de esa voz). Estos embeddings permiten comparar si dos segmentos pertenecen a la misma persona.

3. Similitud coseno y umbral

Para determinar si dos segmentos pertenecen al mismo hablante, se calcula la similitud coseno entre sus embeddings. Si la similitud supera el umbral de 0.85, se consideran el mismo hablante. Este umbral busca un equilibrio entre no confundir hablantes diferentes y no separar demasiado a un mismo hablante.

Como activar la diarizacion

Ve a Ajustes > General > Diarizacion de Interlocutores Activa el interruptor "Identificar hablantes automaticamente". La primera vez que lo actives, AIRecorder descargara el modelo de pyannote necesario (requiere conexion a internet). Una vez descargado, el procesamiento funciona offline. Ajusta el umbral de similitud con el slider: valores mas altos son mas estrictos (menos falsos positivos), valores mas bajos son mas permisivos.

  1. Tienes un proveedor de IA configurado y funcionando. Ve a Ajustes > Agentes IA y pulsa "Verificar modelo".

  3. Si usas Whisper local, asegurate de que Ollama o LM Studio estan corriendo.

Si el problema persiste, revisa los logs en la consola de desarrollo: Help > Toggle Developer Tools.

  3. El modelo de embedding es el correcto.

Si usas un modelo local, asegurate de que tiene suficiente memoria RAM disponible.

Guia para Contribuidores

AIRecorder es un proyecto open source y las contribuciones son bienvenidas. Esta guia te ayudara a empezar rapidamente.

Stack tecnologico

Electron

Framework para aplicaciones de escritorio. Main process (electron/main.js), Preload (electron/preload.js), comunicacion IPC.

React + Vite

Frontend con React, estilos Tailwind CSS, estado global con Redux Toolkit, internacionalizacion con i18next.

Python

Procesamiento de audio y transcripcion con Whisper. Script principal: python/audio_sync_analyzer.py.

Como ejecutar en desarrollo

npm run dev

Este comando inicia el frontend con Vite en modo hot-reload y lanza Electron con la ventana de la app. Cualquier cambio en el codigo React se refleja automaticamente.

Build Frontend

npm run build

Build macOS DMG

npm run electron:build

Convencion de commits

Utilizamos Conventional Commits. Cada commit debe empezar con un tipo seguido de dos puntos:

✨ feat: Nueva funcionalidad
🐛 fix: Correccion de bug
🔧 refactor: Refactorizacion de codigo sin cambiar comportamiento
📝 docs: Cambios en documentacion
🧹 chore: Tareas de mantenimiento, dependencias, configuracion

Reglas del proyecto

📏 Limite de lineas: Archivos de mas de ~300 lineas son code smell. Divide en modulos mas pequenos agrupados por contexto.
🌐 Internacionalizacion: Todo texto visible en la UI debe usar claves de i18n, nunca texto hardcodeado.
🔒 Seguridad: Nunca comitear claves API, tokens o credenciales. Usa variables de entorno o configuracion del usuario.
📖 Documentacion: Si modificas logica en areas clave, actualiza el README correspondiente (ver matriz de mantenimiento en AGENTS.md).
